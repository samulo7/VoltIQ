# 前端产品化改造清单 v1

## 目标
将当前 VoltIQ 前端从“可用的内部管理台”升级为“可对客户演示、可交付、可持续迭代的业务产品前端”。

本清单重点不是单纯美化页面，而是围绕业务链路、角色体验、视觉一致性、交互完整性和 AI 能力展示，建立一套可落地的产品化改造框架。

---

## 一、统一产品基线（P0）

### 1.1 视觉系统统一
先建立一套贯穿全站的设计基线，避免页面各自为战。

- 统一 8pt 间距体系
- 统一圆角、阴影、边框、卡片风格
- 统一表格密度、按钮尺寸、表单间距
- 统一颜色语义：
  - 蓝色：主操作
  - 绿色：成功 / 成单
  - 橙色：提醒 / 跟进中
  - 红色：风险 / 逾期 / 流失
- 统一字体层级：页面标题、区块标题、正文、辅助文本
- 统一图标风格与状态色

### 1.2 状态组件统一
所有业务页面都要补齐标准状态体验。

- 空状态
- 加载骨架
- 错误状态
- 无权限状态
- 成功反馈
- 无数据但可引导下一步的状态

### 1.3 页面结构统一
建议所有核心业务页采用统一页面骨架。

- 页头：标题 + 页面说明 + 主操作
- 筛选区：搜索、筛选、快捷视图
- 主内容区：表格 / 看板 / 图表 / 会话
- 辅助区：详情面板 / 快捷动作 / 提示信息

### 1.4 交互标准统一
所有页面要建立一致的交互标准。

#### 列表页统一能力
- 默认筛选
- 快速搜索
- 常用筛选保存
- 批量操作
- 列配置
- 导出
- 分页体验统一

#### 表单页统一能力
- 必填提示前置
- 错误提示清晰
- 保存成功反馈
- 提交后下一步引导
- 草稿态支持（如适用）

#### 高风险动作统一能力
- 二次确认
- 操作结果反馈
- 失败原因清晰可读

### 1.5 角色化体验统一
当前系统已存在 `operator`、`sales`、`manager` 三类角色，不应只停留在菜单权限差异，而应进一步体现为：

- 首页内容不同
- 默认视角不同
- 关键指标不同
- 快捷入口不同
- 页面默认排序/筛选不同

## 二、按页面的产品化改造清单

## 2.1 首页 / 工作台（P0）

### 当前问题
- 首页价值感弱
- 登录后看不到“今天要做什么”
- 不同角色没有差异化体验

### 产品目标
让用户一进入系统就能看到：
- 当前最重要的工作
- 关键业务指标
- 风险提醒
- 常用入口

### 改造建议

#### operator 工作台
- 今日新增线索
- 待分配线索
- 待生成内容任务
- 知识库会话量
- 最近异常提醒

#### sales 工作台
- 今日待跟进客户
- 本周新增商机
- 即将超期商机
- 本月成交目标完成度
- 我的待办清单

#### manager 工作台
- 团队漏斗概览
- 本周线索→商机→成单转化
- 团队成员排行
- 风险商机预警
- 审计异常提醒

### 必补能力
- KPI 卡片
- 趋势图
- 待办列表
- 快捷入口
- 最近操作流
- 角色化默认布局

---

## 2.2 线索管理（P0）

### 当前问题
- 容易停留在 CRUD 表格层
- 缺少业务优先级判断
- 缺少批量处理效率
- 缺少线索后续动作引导

### 产品目标
将线索页升级为“获客分发中心”。

### 改造建议
- 顶部加入业务摘要卡：
  - 今日新增
  - 未分配
  - 重复待合并
  - 高潜线索
- 中部加入线索分布或漏斗信息
- 列表信息标签化展示：
  - 来源
  - 行业
  - 地区
  - 状态
  - 负责人
- 突出关键字段：
  - 公司
  - 联系人
  - 电话
  - 最近跟进时间
- 对重复线索、高风险线索给出明显标识

### 必补交互
- 批量分配负责人
- 批量改状态
- 批量打标签
- 合并重复线索流程化
- 新建线索成功后引导下一步：
  - 继续录入
  - 分配销售
  - 发起首次跟进

### 详情体验建议
详情不要只靠跳转新页，建议提供抽屉或右侧详情面板，聚合：
- 基本信息
- 跟进记录
- 商机信息
- 内容触达记录
- 审计摘要

---

## 2.3 跟进记录（P1）

### 当前问题
- 容易变成历史日志页
- 看不到今日最重要的待跟进对象
- 跟进后缺少后续推进引导

### 产品目标
将其做成“销售推进工作台”，强调连续性和效率。

### 改造建议
- 页面拆成两块：
  - 今日待跟进
  - 历史跟进记录
- 提供多种视图：
  - 列表视图
  - 时间轴视图
  - 客户维度视图
- 每条跟进记录展示：
  - 跟进方式
  - 跟进结果
  - 下次跟进时间
  - 当前商机阶段

### 必补能力
- 逾期未跟进提醒
- 跟进结果模板
- 最近联系摘要
- 一键设置下次提醒
- 一键转商机
- 一键标记失联/无效

---

## 2.4 商机管理（P0）

### 当前问题
- 如果仅用表格呈现，会缺少推进感
- 缺少阶段管理体验
- 缺少风险提示和概率表达

### 产品目标
将其升级为“销售漏斗驾驶舱”。

### 改造建议
- 顶部增加商机漏斗：
  - 新建
  - 接触中
  - 方案中
  - 谈判中
  - 成单
  - 丢单
- 支持双视图：
  - 表格视图
  - 看板视图（重点建议）
- 商机卡片或列表突出：
  - 客户名称
  - 当前阶段
  - 预计金额
  - 成单概率
  - 最近跟进时间
  - 风险标签

### 必补交互
- 阶段推进动作明确化
- 丢单必须填写原因
- 即将停滞商机提醒
- 超期未推进提醒
- 成单预测与预计回款时间展示

### 推荐增强项
- 商机评分
- 风险预警
- 转化率分析
- 分阶段统计

---

## 2.5 成单记录（P1）

### 当前问题
- 只展示结果，缺少复盘价值
- 对管理层支持不足

### 产品目标
将其升级为“结果沉淀与复盘页”。

### 改造建议
- 顶部摘要卡：
  - 本月成单数
  - 本月成单金额
  - 平均成交周期
  - 成单来源分布
- 列表突出：
  - 成交金额
  - 成交周期
  - 来源线索
  - 所属销售
  - 所属行业

### 必补能力
- 赢单原因沉淀
- 关键动作回溯
- 成交趋势分析
- 销售排行
- 行业/来源分析
- 导出复盘数据

---

## 2.6 内容任务（P1）

### 当前问题
- 容易像普通任务列表
- AI 价值感不强
- 生成、审核、采用之间的链路不完整

### 产品目标
将其升级为“AI 内容生产工作台”。

### 改造建议
- 顶部摘要卡：
  - 今日生成数
  - 待处理任务数
  - 待审核内容数
  - 热门模板使用量
- 建立任务流转状态：
  - 新建
  - 生成中
  - 待审核
  - 已采用
  - 已废弃
- 内容列表或卡片中展示：
  - 内容类型
  - 场景
  - 目标客群
  - 生成时间
  - 状态

### 必补能力
- 左侧任务列表 + 右侧预览/编辑
- 一键复制
- 二次编辑
- 模板中心
- 版本对比
- 生成结果评分
- 采用率反馈闭环

---

## 2.7 智能客服（P1）

### 当前问题
- 纯聊天视图容易像 demo
- 回答可信度展示不足
- 缺少运营视角

### 产品目标
将其升级为“可运营、可追溯、可演示的企业智能客服台”。

### 改造建议
- 页面采用三栏结构：
  - 左：会话列表
  - 中：聊天主区域
  - 右：知识来源 / 命中片段 / 会话信息
- 强化 AI 回答可信展示：
  - 每条回答展示来源
  - 来源支持展开查看
  - 无命中时明确提示
- 会话列表突出：
  - 会话主题
  - 最近提问时间
  - 满意度
  - 是否人工介入

### 必补能力
- 来源引用 UI 强化
- 快捷问题推荐
- 输入建议词
- 会话筛选
- 未解决问题标记
- 热门问题与未命中问题分析

---

## 2.8 数据看板（P0）

### 当前问题
- 很多看板只有图，没有业务解释
- 缺少行动导向

### 产品目标
让销售和管理层快速判断当前业务状态，并知道下一步重点。

### 改造建议
- 顶部 KPI：
  - 线索总量
  - 跟进率
  - 商机转化率
  - 成单金额
  - 环比变化
- 中部图表：
  - 漏斗图
  - 趋势图
  - 来源分布
  - 团队排行
- 下部洞察区：
  - 本周异常点
  - 高潜客户提醒
  - 转化瓶颈阶段
  - 建议动作

### 必补能力
- 时间范围切换
- 团队 / 个人切换
- 图表 hover 解释
- 导出截图/PDF
- 环比同比标记

---

## 2.9 审计日志（P2）

### 当前问题
- 容易成为纯技术日志页
- 风险感知弱
- 快速定位异常困难

### 产品目标
将其升级为“企业级审计与追责支撑页”。

### 改造建议
- 顶部摘要：
  - 今日操作总数
  - 敏感操作数
  - 失败操作数
  - 高风险行为数
- 列表强化字段：
  - 操作人
  - 模块
  - 动作
  - 目标对象
  - 时间
  - IP / 结果
- 敏感操作高亮显示
- 失败原因支持展开查看

### 必补能力
- 按用户/模块/时间过滤
- 敏感操作筛选
- 变更前后对比
- 上下文信息抽屉

---

## 三、分阶段优先级建议

### 第一批必须做（P0）
1. 首页工作台
2. 线索管理
3. 商机管理
4. 数据看板
5. 全局视觉与交互基线

这一批完成后，整体产品气质会从“开发态后台”提升为“可演示产品”。

### 第二批体验增强（P1）
1. 跟进记录
2. 成单记录
3. 内容任务
4. 智能客服

这一批完成后，业务链路会更完整，AI 价值也更容易被客户感知。

### 第三批企业级补强（P2）
1. 审计日志
2. 配置页、帮助页、空状态文案体系

---

## 四、最容易立刻提升质感的 10 个动作

1. 所有页面补齐统一页头说明
2. 首页改造成角色化工作台
3. 线索页顶部加入业务摘要卡
4. 商机页增加看板 + 表格双视图
5. 全站统一空状态、加载状态、错误状态
6. 统一按钮层级与文案风格
7. 所有表格页补齐筛选、批量操作、列配置
8. 详情展示从“跳转页”升级为“抽屉/侧边详情面板”
9. 数据看板补齐环比、异常提醒、洞察结论
10. 智能客服页强化来源引用与可信展示

---

## 五、后续可继续细化的方向

如果进入实施阶段，建议下一步继续拆成以下产物：

1. 按页面输出组件级改造清单
2. 输出统一设计规范（色板、间距、卡片、表格、按钮、状态）
3. 输出页面优先级排期方案（按 2 周 / 4 周节奏）
4. 输出“客户演示路径”设计（operator / sales / manager 三条演示线）
5. 输出首页、线索页、商机页、看板页的高保真原型建议
